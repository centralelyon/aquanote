Générateur de Vue de Dessus pour Natation

Ce script permet de générer une vue de dessus d'une seule ligne de natation à partir d'une vidéo latérale en utilisant l'homographie.

Fichiers inclus

  • video_vues_de_dessus_une_ligne.py - Script principal
  • test_vue_dessus.py - Script de test et exemples
  • exemple_calibration.json - Fichier JSON d'exemple pour la calibration
  • README_vue_dessus.md - Ce fichier de documentation

Installation des dépendances

pip install opencv-python numpy

Ou utilisez le script de test pour générer un fichier requirements.txt :

python test_vue_dessus.py --requirements
pip install -r requirements.txt

Utilisation

Utilisation basique

python video_vues_de_dessus_une_ligne.py video_course.mp4 calibration.json

Options avancées

# Spécifier une ligne et un fichier de sortie
python video_vues_de_dessus_une_ligne.py video_course.mp4 calibration.json -l 3 -o ligne3_dessus.mp4
# Aperçu des données de calibration
python video_vues_de_dessus_une_ligne.py video_course.mp4 calibration.json --preview
# Aide complète
python video_vues_de_dessus_une_ligne.py --help

Traitement de toutes les lignes

# Windows PowerShell
for ($i=1; $i -le 8; $i++) {
python video_vues_de_dessus_une_ligne.py course.mp4 calibration.json -l $i -o "ligne${i}_dessus.mp4"
}
# Linux/Mac
for i in {1..8}; do
python video_vues_de_dessus_une_ligne.py course.mp4 calibration.json -l $i -o ligne${i}_dessus.mp4
done

Structure du fichier JSON de calibration

Le fichier JSON doit contenir les points de calibration qui permettent de faire la correspondance entre les coordonnées de la vidéo et les coordonnées réelles de la piscine.

{
"distance": 50,
"style": "freestyle",
"lignes": {
"ligne1": "Nageur 1",
"ligne2": "Nageur 2",
"ligne3": "Nageur 3",
"ligne4": "Nageur 4",
"ligne5": "Nageur 5",
"ligne6": "Nageur 6",
"ligne7": "Nageur 7",
"ligne8": "Nageur 8"
},
"videos": [
{
"name": "camera_fixe_droite",
"width": 1920,
"height": 1080,
"srcPts": [
[100, 200], // Coin haut-gauche dans la vidéo
[1800, 180], // Coin haut-droit dans la vidéo
[1850, 900], // Coin bas-droit dans la vidéo
[50, 920] // Coin bas-gauche dans la vidéo
],
"destPts": [
[0, 0], // Coin haut-gauche de la piscine (0m, 0m)
[50, 0], // Coin haut-droit de la piscine (50m, 0m)
[50, 20], // Coin bas-droit de la piscine (50m, 20m)
[0, 20] // Coin bas-gauche de la piscine (0m, 20m)
],
"start_side": "right",
"one_is_up": true
}
]
}

Conseils pour la calibration

  1. Précision des points : Les srcPts doivent correspondre exactement aux coins de la piscine dans la vidéo
  2. Coordonnées réelles : Les destPts représentent les coordonnées réelles de la piscine en mètres
  3. Ordre des points : L'ordre des points doit être cohérent (sens horaire ou anti-horaire)
  4. Qualité de la transformation : Plus les points sont précis, meilleure sera la transformation homographique

Fonctionnalités

  • Extraction d'une ligne spécifique : Choisissez quelle ligne extraire (1-8)
  • Transformation homographique : Conversion automatique vers une vue de dessus
  • Résolution configurable : Ajustement de la résolution en pixels par mètre
  • Interface en ligne de commande : Utilisation simple avec arguments
  • Mode aperçu : Vérification des données de calibration avant traitement
  • Gestion d'erreurs : Vérifications complètes des fichiers et paramètres

Algorithme

  1. Chargement des données : Lecture du fichier JSON de calibration
  2. Ajustement des points : Adaptation des coordonnées pour la ligne sélectionnée
  3. Calcul d'homographie : Utilisation de cv2.findHomography() avec RANSAC
  4. Transformation des frames : Application de cv2.warpPerspective() sur chaque frame
  5. Extraction de ligne : Découpage de la région correspondant à la ligne
  6. Génération vidéo : Création de la vidéo de sortie avec les frames transformées

Test

Utilisez le script de test pour vérifier le fonctionnement :

python test_vue_dessus.py

Pour voir tous les exemples d'utilisation :

python test_vue_dessus.py --help

Paramètres techniques

  • Résolution par défaut : 20 pixels par mètre
  • Format de sortie : MP4 (codec mp4v)
  • Préservation du FPS : Même cadence que la vidéo d'entrée
  • Largeur de ligne : Largeur standard + 50% pour inclure les bords

Troubleshooting

Erreur "Impossible de calculer la matrice d'homographie"

  • Vérifiez que les 4 points de calibration ne sont pas colinéaires
  • Assurez-vous que les points correspondent bien aux coins de la piscine

Vidéo de sortie déformée

  • Vérifiez l'ordre des points dans srcPts et destPts
  • Assurez-vous que les coordonnées destPts sont en mètres

Performance lente

  • Réduisez la résolution de la vidéo d'entrée
  • Ajustez le paramètre pixels_per_meter pour une résolution plus faible
